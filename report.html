Scope

四个核心模块，一层任务编排

真正的业务能力集中在四个文件；入口文件只抽取调度、队列和数据库状态机。函数“返回”不总是 Python 的 return：大量结果写入 Anime 实例字段或全局进度表。

A

Anime

从页面/API 元数据一路推进到 m3u8、下载、通知与 FTP，是主业务聚合对象。

C

Config

配置正规化、版本迁移、订阅清单、Cookie 生命周期和共享任务进度。

D

Danmu

拉取弹幕与在线屏蔽词，将位置、颜色和时间换算为 ASS Dialogue。

L

ColorPrint

统一控制台颜色与按日落盘日志，被所有业务路径作为横切能力调用。

报告边界：不展开 argparse → __cui() 的命令行选项分发，也不展开 Dashboard/Server.py 的 HTTP/WebSocket 路由；但保留它们共同依赖的 build_anime / check_tasks / worker / SQLite 核心编排契约。

表面很薄，状态很深

CLI 和 Web 都把任务转换为 SN；之后进入同一套有状态对象。数据库与文件系统不是外围细节，而是决定重试、上传和“是否已完成”的事实来源。

Call chains

八条真实业务调用链

展开任意链查看分支条件与数据落点。箭头表示直接调用；方括号表示条件分支或外部 I/O。

01

配置启动链

从不可信 JSON 到带运行时字段的 settings
Config.read_settings(config='')→__read_settings_file()→[__update_database]→[__update_settings]→__remove_superfluous_logs()
  • 补充 working_dir / aniGamerPlus_version / use_gost 三个运行时键；write_settings() 落盘前再删除。
  • 正规化端口、并发、路径、封装格式和下载模式；可能改写配置或创建默认配置。
02

番剧识别链

一个 SN 派生出整个剧集映射
build_anime(sn)→Anime(sn)→__get_src()→__get_title()→__get_episode()→__get_bangumi_name()→__get_episode_list()
  • Web 源为 BeautifulSoup；移动源为 JSON dict。无动画时直接 sys.exit(1)。
  • 剧集映射统一为 dict[str, int]，兼容电影、特别篇、中文配音与单集页面。
03

播放授权链

device、token、广告、master playlist
get_m3u8_dict()→__get_m3u8_dict()→get_device_id()→gain_access()→[unlock/check_lock/ad]→get_playlist()→parse_playlist()
  • 所有 HTTP 基本都穿过 __request()，所以重试、Cookie 刷新、代理与日志逻辑集中在这里。
  • 输出缓存进 _m3u8_dict；第二次读取不会重新握手，除非实例重建。
04

分段下载链

自己拉 TS，再调用 ffmpeg 合并
download(...)→__segment_download_mode(resolution)→download_chunk(uri) × N→[localize m3u8]→ffmpeg -c copy→move/copy
  • 每个 chunk 经 __request(max_retry=segment_max_retry)；Semaphore 限制单视频的分段并发。
  • 进度写入 Config.tasks_progress_rate[sn]['rate']；完成后设置路径、文件名和 MB 大小。
05

ffmpeg 直拉链

远端 m3u8 直接交给子进程
download(...)→__ffmpeg_download_mode(resolution)→Popen(ffmpeg)+check_ffmpeg_alive()→move/copy
  • 守护线程每分钟检查临时文件是否至少增长 3 MiB；否则 kill ffmpeg。
  • 成功条件为 returncode=0 且 stderr 不含 Failed to open segment。
06

弹幕生成链

JSON 弹幕变成可随片播放的 ASS
Anime.download()→Danmu(sn, ass_path, cookies)→download(ban_words)→get_BGRcolor / find_ban_word→[write ASS]
  • 本地屏蔽词列表会被在线屏蔽词原地追加；滚动弹幕按可用轨道和随机 10–14 秒速度排布。
  • 方法显式返回 None；成功与否通过文件和日志观察。
07

FTP 断点续传链

识别 Pure-FTPd 临时文件并校验字节数
worker()→Anime.upload(tag)→connect_ftp()→transfercmd(STOR, offset)→[size verify]→rename()
  • 输入依赖实例先拥有 local_video_path；也可用 debug_file 覆盖。
  • 最终返回 bool 并同步设置 upload_succeed_flag。
08

订阅与状态机链

清单 → 判重 → 下载 → 可选上传 → 回写
Config.read_sn_list()→check_tasks()→read_db / insert_db→queue→worker()→Anime.download()→[upload]→update_db()
  • all / latest / largest-sn / single 只影响挑选哪些 episode SN，后续 worker 完全相同。
  • 下载与上传分别由 Semaphore 限流；下载槽位要等冷却计时结束才释放。

Engineering notes

阅读调用链后最值得记住的事

这些不是对功能的否定，而是调用者、维护者和重构者最容易踩到的契约边角。

成功由副作用表达

download() 无布尔返回；核心编排以 video_size > 5 判断成功。直接集成时不能只检查异常或返回值。

异常层级非常规

TryTooManyTimeError 继承 BaseException 而非 Exception；常见的 except Exception 接不住它。

存在进程级可变状态

Config.cookie、tasks_progress_rate、队列与信号量都是共享可变对象；函数签名没有完整表达其依赖。

HTTP 头可能跨请求残留

__request() 令 current_header = self._req_header 后原地合并 addition_header，附加头可能留给同实例后续请求。

部分失败直接结束进程/线程

无效 SN、地区限制、VIP 锁定、广告校验等路径会 sys.exit(1)，不是统一异常或错误值。

get_local_ip 异常分支有缺陷

连接失败时源码调用 local_ip.close()，但该变量尚未赋值；预期应关闭 socket 并提供回退值。

debug_mode 不是“多打印日志”

Anime(debug_mode=True) 会跳过代理、请求头和全部元数据抓取，得到的是未准备完成的壳对象。

ban_words 会被修改

Danmu.download() 将在线屏蔽词 append 到传入列表。若直接传配置中的列表，同一进程多次调用会累积条目。
